## Summary

Added `WARN_PRINT_ONCE_ED` warnings when `Geometry2D::triangulate_polygon()` returns an empty index array in `Polygon2D::_notification()`. Previously, if polygon data was invalid, self-intersecting, or degenerate, the polygon would silently render nothing with no diagnostic output.

## Changes Made

- **`scene/2d/polygon_2d.cpp`**: Added `WARN_PRINT_ONCE_ED` calls after both triangulation sites:
  1. The main polygon path (`if (invert || polygons.is_empty())`) — warns when full polygon fails triangulation
  2. The individual sub-polygons path (`else` branch loop) — warns and `continue`s when a sub-polygon fails triangulation

## Notes

- There is an existing open PR (#118057) by Omniologist covering only the first triangulation site. This PR extends coverage to both call sites.
- The fix is purely additive — no behavioral changes for valid polygons.
- `WARN_PRINT_ONCE_ED` is the correct macro (editor-only in release, shows once to avoid spam).
- Full Godot build was not run (the change is trivially correct — no new dependencies, no logic changes).
